HARD RULES (from CLAUDE.md)
--------------------------------------------------------------------------------
1. Git account: `gh auth switch --user abedubas-alchemydev --hostname github.com`
   then `gh auth status`.
2. Zero AI attribution in commits or PRs. After every commit:
   `git log -1 --pretty=full` — verify clean.
3. Stage files by name. NEVER `git add -A`.
4. NEVER `--no-verify`, never skip hooks. Failed hook → new commit (never `--amend`).
5. Frontend talks to backend ONLY through `/api/backend/...` (BFF proxy),
   `credentials: 'include'`, via `apiRequest` / `buildApiPath` from `lib/api.ts`.
   Do NOT add Authorization/JWT headers client-side.
6. Do NOT touch `lib/auth.ts`, the BFF proxy, admin user-management routes,
   or `lib/email.ts` unless the plan explicitly calls for it.
7. UI conventions: Tailwind only, `lib/format.ts` for formatting, Lucide icons,
   (app) group wrapped by `components/layout/app-shell.tsx`.
8. Long output → `reports/` or `plans/`. Chat reply ≤ 5 lines.
